# Pilot Decision Outcome Guide

## GO para piloto
Usar quando:
- criterios obrigatorios estiverem verdes
- risco residual for baixo ou moderado controlado
- revisao clinica final estiver concluida
- simulador estiver estavel nos cenarios prioritarios

### Acao seguinte
- registrar GO formal
- definir escopo do piloto
- definir usuarios do piloto
- iniciar coleta estruturada de feedback

## GO para piloto com restricoes
Usar quando:
- produto estiver suficientemente seguro para piloto controlado
- houver pendencias nao bloqueantes claramente delimitadas
- as restricoes puderem ser comunicadas e monitoradas

### Acao seguinte
- registrar restricoes do piloto
- delimitar o que nao deve ser usado no piloto
- monitorar de perto os modulos com ressalva

## HOLD
Usar quando:
- houver bug bloqueante
- houver bug alto em modulo central
- revisao clinica final nao estiver fechada
- risco editorial ou tecnico ainda for relevante

### Acao seguinte
- registrar motivo do HOLD
- listar correcoes obrigatorias
- retornar para estabilizacao ou revisao clinica conforme o caso
