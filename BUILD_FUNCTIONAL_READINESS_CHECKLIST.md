# Build Functional Readiness Checklist

## O projeto pode ser chamado de build funcional quando
- existe estrutura real de codigo do app no repositorio
- o app abre sem crash
- a Home funciona
- a navegacao principal funciona
- o modulo central carrega
- o simulador responde a alteracao de parametros
- baseline funciona
- reset funciona
- fallback funciona
- nao ha erro critico no fluxo principal
- disclaimer educacional esta visivel
- nao ha uso de dados reais
- nao ha recomendacao autonoma de conduta

## Resultado
Se todos os itens acima forem verdadeiros, o app ja entrou em fase de build funcional de teste interno.
