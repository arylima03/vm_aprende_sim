# Next Phase Paths Guide

## 1. Nova validacao controlada
Usar quando:
- V1.1 corrigiu pontos obrigatorios
- ainda se deseja confirmar usabilidade e valor educacional com novo grupo pequeno
- ha interesse em observar se os refinamentos resolveram as friccoes do piloto

### Objetivo
Confirmar se a V1.1 realmente resolveu os principais problemas do piloto fechado.

## 2. Ampliacao controlada do piloto
Usar quando:
- V1.1 estiver estavel
- risco residual for baixo ou moderado controlado
- feedback anterior tiver sido favoravel
- a equipe quiser aumentar ligeiramente o numero de usuarios sem perder controle

### Objetivo
Ganhar mais volume de uso mantendo controle e rastreabilidade.

## 3. Manutencao com backlog para v1.2
Usar quando:
- nao houver necessidade imediata de ampliar validacao
- o produto estiver util para contexto interno controlado
- o foco for consolidar backlog e planejar uma proxima iteracao mais ampla

### Objetivo
Evitar expandir cedo demais e preparar uma proxima versao com mais maturidade.

## Regra de escolha
- quando houver duvida, preferir nova validacao controlada antes de ampliacao mais ampla
- so considerar ampliacao controlada se simulador e fluxos centrais estiverem claramente estaveis
