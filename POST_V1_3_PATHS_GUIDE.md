# Post V1.3 Paths Guide

## 1. Manutencao estavel renovada
Usar quando:
- V1.3 entregou o backlog prioritario
- nao ha urgencia de nova iteracao maior
- backlog remanescente e secundario
- produto continua estavel e util

### Objetivo
Preservar o valor validado do produto com monitorizacao leve e backlog organizado.

## 2. Backlog para v1.4
Usar quando:
- V1.3 foi bem sucedida, mas surgiram melhorias suficientes para justificar nova versao incremental
- ha ganho claro em organizar os ajustes em pacote proprio
- ainda nao faz sentido ampliar risco, escopo ou uso sem esses refinamentos

### Objetivo
Consolidar uma nova versao incremental sem perder controle do produto.

## 3. Nova evolucao controlada
Usar quando:
- houver oportunidade clara de evolucao incremental relevante
- risco residual permanecer baixo ou moderado controlado
- a equipe decidir por um novo ciclo pequeno e delimitado

### Objetivo
Evoluir o produto gradualmente sem romper o modelo de seguranca e controle adotado ate aqui.

## Regra de escolha
- quando houver duvida, preferir manutencao estavel renovada ou backlog para v1.4 antes de nova evolucao
- so considerar nova evolucao controlada se houver justificativa clara de valor adicional
