# Sprint 2 Entry Criteria

## Objetivo
Definir criterios objetivos para iniciar a Sprint 2 sem carregar problemas da Sprint 1.

## Entrada obrigatoria
- Sprint 1 auditada
- status minimo: aprovado para Sprint 2 ou aprovado com ressalvas controladas
- Firebase funcional
- FlutterFlow conectado ao banco correto
- modules, scenarios, asynchronies e quizzes visiveis no app

## Nao iniciar Sprint 2 se houver
- HomePage sem carregar modules
- queries quebradas em listas principais
- bindings errados em detalhes de cenario ou assincronia
- problema de acesso ao Firebase correto
- rules bloqueando leitura legitima do app

## Preparos especificos para Sprint 2
- quizzes validados no Firestore
- simulator_profiles importados e revisados
- defaults por cenario revisados
- asset naming convention revisada
- waveform state keys revisadas

## Evidencias minimas antes de iniciar
- print da Home carregada
- print da lista de cenarios
- print da lista de assincronias
- print do Firestore com simulator_profiles
- confirmacao de que o projeto FlutterFlow aponta para o Firebase dev certo

## Go decision
- GO: iniciar Sprint 2
- HOLD: corrigir pendencias de estrutura antes de seguir
