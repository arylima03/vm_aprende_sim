# Simulator Asset Production Checklist

## Objetivo
Garantir consistencia visual e tecnica dos assets de curvas da v1.

## Para cada estado
- criar asset de pressao
- criar asset de fluxo
- criar asset de volume
- confirmar nome conforme convencao
- revisar legibilidade em web e iPad
- revisar coerencia didatica com a interpretacao

## Validacoes
- nome do asset bate com simulator_profile
- estado possui tres assets principais
- baseline existe para cada profile
- assets especiais de assincronia sao distinguiveis do baseline

## Boas praticas
- usar visual limpo
- manter escala visual coerente entre estados do mesmo modo
- evitar excesso de detalhe grafico
- priorizar clareza pedagogica

## Antes de publicar
- testar troca de asset ao mudar sliders
- testar fallback baseline
- revisar assets em iPhone, iPad e web
